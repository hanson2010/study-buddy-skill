# 作业批改工作流（eval_workflow）

本文件定义通用作业批改的处理流程，支持多种题型和场景，在必要时调用 conversion_workflow 和 essay_workflow。

---

## 一、适用场景

- **作业批改**：用户上传作业照片进行批改
- **试卷批改**：用户上传试卷照片进行批改
- **错题订正**：用户请求订正已有的错题
- **作文批改**：用户上传作文照片进行批改（调用 essay_workflow）

---

## 二、处理流程

### 1. 输入识别与路由

**输入类型识别**：
- **图片输入**：用户上传作业/试卷/作文照片
- **文字输入**：用户直接输入题目内容
- **混合输入**：用户上传图片并附带文字说明

**内容类型识别**：
- **作业/试卷**：包含选择题、填空题、解答题、计算题、证明题等多种题型
- **作文**：包含标题、段落、正文等作文结构
- **其他**：标准答案、范文等参考材料

**路由逻辑**：
```
用户输入
    │
    ├── 图片输入
    │       │
    │       ├── 识别为作文 → 调用 conversion_workflow → 调用 essay_workflow
    │       │
    │       └── 识别为作业/试卷 → 调用 conversion_workflow → 执行通用批改流程
    │
    └── 文字输入
            │
            ├── 识别为作文 → 执行 essay_workflow 中的批改流程
            │
            └── 识别为作业/试卷 → 执行通用批改流程
```

**未支持学科处理**：本技能仅支持六科（语文、数学、英语、物理、化学、生物）。识别作业/试卷所属学科后，若为未支持学科，简单跳过即可，不进行批改处理；用户询问时，可告知不支持该学科。

### 2. 图片处理（如适用）

当用户上传图片时，调用 [conversion_workflow.md](conversion_workflow.md) 进行处理：

- **日期确定**：根据图片内容识别日期或使用当前时间
- **图片存储**：保存到 `raw/YYYY/MM/` 目录，存储时机为文件接收后立即存储，且只存储一次；拷贝前检查是否已存在同名文件，若已存在则跳过拷贝，直接使用已存在的文件路径
- **OCR 识别**：将图片内容转换为文字
- **内容结构化**：提取题目类型、知识点、答案等信息

### 3. 题目解析与批改

**题目类型识别**：
- **选择题**：识别题目描述、选项、用户答案
- **填空题**：识别题目描述、填空位置、用户答案
- **解答题**：识别题目描述、用户解答过程、最终答案
- **计算题**：识别已知条件、计算过程、最终答案（含单位）
- **证明题**：识别已知条件、求证结论、证明过程
- **判断题**：识别题目描述、用户判断

**批改流程**：
1. **题目理解**：分析题目要求和解题思路
2. **答案判断**：对比用户答案与正确答案
3. **错误分析**：归类错误类型（概念不清、计算失误、审题错误、方法不当）
4. **解析生成**：提供详细解答步骤和思路
5. **知识点标注**：标注涉及的知识点和难度等级

### 4. 作文批改（如适用）

当识别为作文时，**由 eval_workflow 负责图片处理和 OCR 识别**，然后调用 [essay_workflow.md](essay_workflow.md) 进行批改：

**图片处理（由 eval_workflow 负责）**：
- 调用 conversion_workflow 进行图片存储和 OCR 识别
- 将图片内容转换为文字
- 提取作文标题、段落、字数等信息

**传递给 essay_workflow 的数据**：
- `title`：作文标题
- `paragraphs`：段落内容数组
- `word_count`：字数统计
- `essay_type`：作文类型
- `image_path`：图片存储路径（如有）

**作文批改流程**：
- **审题分析**：分析作文题目和立意
- **分项评分**：按高考评分标准分项评分（立意、结构、语言、素材、创新）
- **问题诊断**：指出作文存在的问题
- **升格建议**：提供具体的改进建议
- **范文对照**：对比同类范文
- **片段练习**：生成针对性练习

### 5. 错题归档

**归档条件**：
- 题目答案错误
- 解答过程有明显错误
- 用户主动要求归档

**归档流程**：
1. **文件生成**：创建 `subjects/<学科>/YYYY/MM/<YYYY-MM-DD>-<descriptive-slug>.md` 文件
2. **Frontmatter 填写**：包含学科、知识点、难度、错误类型等信息
3. **内容写入**：写入题目描述、用户答案、正确答案、解析、图片引用等
4. **索引更新**：更新对应学科 `_index.md` 的薄弱点表和正确率

**Frontmatter 模板**：
```yaml
---
date: YYYY-MM-DD
subject: [学科]
topic: [知识点]
difficulty: medium  # easy/medium/hard
question_type: [题目类型]
error_type: [错误类型]
is_weak_area: false  # runtime-derived
image_path: [图片路径，如有]
correct_rate: [正确率]
---
```

### 6. 智能辅导六步法（错题辅导与交互确认）

对错误题目执行「智能辅导六步法」，并进行交互式断点控制：

1. **诊断/识别**：图片识别并判定对错，确认薄弱知识点
2. **错因分析**：深入分析错误原因，归类错因（概念/计算/审题/方法等）
3. **视频推荐**：在分步讲解前，先推荐 1-2 个国内平台的概念/方法类微课视频或提供搜索关键词。
4. **分步讲解**：分步进行解答，并使用图形、图表辅助讲解或进行原图标注。
   - **交互断点**：分步讲解完成后，**必须暂停并主动向用户确认是否理解、有无疑问**，严禁未经用户确认直接输出练习题目。
5. **举一反三**：在用户做好练习准备后，生成三层难度梯度的练习题，提供答案和易错提醒。
6. **归档/同步**：错题归档并更新正确率与学情数据。

### 7. 相似错题检测

**检测流程**：
1. **检索错题库**：搜索 `subjects/<学科>/YYYY/MM/` 下的同类错题
2. **相似度匹配**：根据知识点、题目类型、错误类型进行匹配
3. **复习提醒**：发现相似错题时提醒用户复习

### 8. 闪卡复习

**闪卡生成**：
- **正面**：题目描述和用户错误答案
- **背面**：正确答案、解析、知识点

**复习方式**：
- 间隔重复复习
- 根据薄弱点优先级排序

### 9. 操作概要记入 log 文件

每次执行作业批改流程后，必须将操作概要记入 `output/YYYY/MM/YYYY-MM-DD-log.md` 文件：

**记录内容**：
- **操作类型**：作业批改/试卷批改/作文批改/错题订正
- **处理的文件**：上传的图片路径、识别的题目内容
- **生成的文件**：创建的错题归档文件路径
- **更新的索引**：更新的 `_index.md` 文件及其内容摘要
- **核心数据**：批改题目数量、正确率、新增薄弱点数量

**记录格式**：
```markdown
## 操作记录

### YYYY-MM-DD HH:MM:SS
- **操作类型**：作业批改
- **处理文件**：raw/2026/07/photo.jpg
- **生成文件**：subjects/数学/2026/07/2026-07-12-derivative-applications.md
- **更新索引**：subjects/数学/_index.md（更新薄弱点：函数与导数）
- **核心数据**：批改5题，正确率80%，新增1个薄弱点
```

## 三、与其他 Workflow 的关系

### 调用关系

```
eval_workflow (通用作业批改)
    │
    ├── conversion_workflow (图片处理与 OCR)
    │       └── 图片存储到 raw/YYYY/MM/
    │
    └── essay_workflow (作文批改)
            └── 作文特定的批改流程
```

### 数据传递

- **从 conversion_workflow 接收**：OCR 识别结果、图片存储路径、内容结构化数据
- **传递给 essay_workflow**：作文文本、图片路径、学科信息
- **输出**：批改结果、错题归档文件、薄弱点更新

## 四、使用示例

### 输入示例1：上传作业照片
```text
帮我批改这份数学作业。
[上传图片]
```

### 输入示例2：上传作文照片
```text
帮我批改这篇作文。
[上传图片]
```

### 输入示例3：直接输入题目
```text
已知函数 f(x) = x^2 - 2x + 1，求 f(x) 的最小值。我算出来是 -1，对吗？
```

### 输入示例4：订正错题
```text
帮我订正这道导数的题目。
```

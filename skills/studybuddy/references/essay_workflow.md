# 语文作文批改提升 Workflow

本文件定义语文作文批改的独立流程，不套用通用的「难题破解五步法」。

---

## 前置条件

作文批改流程**不处理图片**，图片处理与 OCR 识别由 [eval_workflow.md](eval_workflow.md) 负责。

### 调用关系

```
用户上传作文图片 → [eval_workflow.md] 图片处理与 OCR 识别 → [essay_workflow.md] 作文批改
```

### 输入要求

进入作文批改流程前，必须满足以下条件：

1. **作文文本已获取**：通过 OCR 识别或用户手动输入获得完整的作文文本
2. **内容完整性**：标题和正文内容完整，无明显缺失
3. **图片引用记录**：`image_path` 已正确记录（如有），方便回溯查看手写原稿

### 输入数据格式

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| `title` | 作文标题 | 是 |
| `paragraphs` | 段落内容数组 | 是 |
| `word_count` | 字数统计 | 是 |
| `essay_type` | 作文类型（argumentation/narration/application） | 是 |
| `image_path` | 图片存储路径（如有） | 否 |

## 一、高考作文评分标准（参考）

| 维度 | 满分 | 评分要点 |
|------|------|----------|
| **立意** | 20分 | 审题准确、立意深刻、中心明确 |
| **结构** | 15分 | 结构清晰、逻辑严密、过渡自然 |
| **语言** | 15分 | 表达流畅、用词准确、文采斐然 |
| **素材** | 15分 | 素材新颖、运用恰当、论据充分 |
| **创新** | 15分 | 观点独到、论证新颖、构思巧妙 |

## 二、作文批改流程

### 1. 审题分析
- **题目解读**：分析题目要求、关键词、限制条件
- **立意判断**：判断学生立意是否准确、是否偏离题意
- **角度评估**：评估选择的写作角度是否新颖、深刻

### 2. 分项评分
- 根据高考评分标准，对立意、结构、语言、素材、创新五个维度分别评分
- 给出总分和评分等级（一类文/二类文/三类文/四类文）
- 标注扣分点和理由

### 3. 问题诊断
- **审题偏差**：是否偏离题意、关键词理解错误
- **结构问题**：开头结尾是否有力、段落逻辑是否清晰
- **语言问题**：表达是否流畅、用词是否准确、是否有病句
- **素材问题**：素材是否陈旧、运用是否恰当、论据是否充分
- **论证问题**：论证是否单一、逻辑是否严密、是否有说服力

### 4. 升格建议
- **针对性修改**：针对每个问题给出具体的修改建议
- **段落重写**：挑选关键段落进行示范改写
- **素材替换**：推荐更合适的素材或事例
- **结构优化**：建议调整文章结构和段落顺序

### 5. 范文对照
- **范文选择**：从 `subjects/语文/` 中查找同题或同类题目的 `model_essay`（作文范文）和 `classmate_essay`（同学范文）
- **对比分析**：对比学生作文与范文的差距，分析范文的优点
- **借鉴建议**：建议学生借鉴范文的哪些方面

### 6. 片段练习
- **针对性练习**：针对薄弱点生成 1-2 个片段写作练习
- **审题练习**：给出类似题目进行审题训练
- **素材积累**：推荐相关素材和事例供学生积累

## 三、作文文件 Frontmatter 模板

```yaml
---
date: YYYY-MM-DD
subject: chinese
topic: 作文
reference_type:  # 空或 standard_answer/model_essay/classmate_essay
essay_type: argumentation  # argumentation（议论文）/ narration（记叙文）/ application（应用文）
title: [作文标题]
word_count: [字数]
image_path:  # OCR识别的原始图片路径，如 raw/YYYY/MM/photo.jpg
score: [总分]
grading:
  conception: [立意得分]/20
  structure: [结构得分]/15
  language: [语言得分]/15
  material: [素材得分]/15
  innovation: [创新得分]/15
grade_level: [一类文/二类文/三类文/四类文]
weak_points:
  - [薄弱点1]
  - [薄弱点2]
---
```

## 四、与 _index.md 的集成规则

### 作文薄弱点记录
语文 `_index.md` 的薄弱点表中，作文相关薄弱点按以下维度记录：

| 知识点 | 错误次数 | 最近错误日期 | 状态 |
|--------|----------|--------------|------|
| 审题偏差 | [N] | [YYYY-MM-DD] | ⚠️ 待巩固 |
| 素材陈旧 | [N] | [YYYY-MM-DD] | ⚠️ 待巩固 |
| 论证单一 | [N] | [YYYY-MM-DD] | ⚠️ 待巩固 |
| 语言平淡 | [N] | [YYYY-MM-DD] | ✅ 已掌握 |

### 更新时机
- **每次作文批改后**：更新语文 `_index.md` 的薄弱点表（新增或更新作文相关薄弱点）
- **每次作文复习后**：连续 3 次同类问题改善，状态从 ⚠️ 待巩固 改为 ✅ 已掌握

## 五、范文对照机制

### 范文来源
1. **model_essay（作文范文）**：老师提供的满分作文、优秀范文
2. **classmate_essay（同学范文）**：同学的高分作文

### 范文使用
- 批改时主动检索同题或同类题目的范文
- 在「范文对照」环节对比分析学生作文与范文的差距
- 推荐学生借鉴范文的优点

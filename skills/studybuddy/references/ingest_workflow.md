# 学习资料导入工作流（ingest_workflow）

本文件定义学习资料导入的处理流程，支持多学科资料的分类导入和归档。

---

## 一、适用场景

- **学习资料导入**：用户上传学习资料（课本照片、讲义、笔记、知识点总结等）
- **多学科资料导入**：一次导入包含多个学科的资料
- **参考材料导入**：导入标准答案、范文、教学大纲等参考材料
- **知识点资料导入**：导入特定知识点的讲解资料

---

## 二、处理流程

### 1. 输入识别与路由

**输入类型识别**：
- **图片输入**：用户上传资料照片（课本页面、讲义、笔记等）
- **文字输入**：用户直接输入资料内容
- **混合输入**：用户上传图片并附带文字说明

**内容类型识别**：
- **学习资料**：课本内容、讲义、笔记、知识点总结
- **参考材料**：标准答案、范文、教学大纲
- **多学科资料**：包含多个学科的综合资料

**路由逻辑**：
```
用户输入
    │
    ├── 图片输入
    │       │
    │       └── 识别资料内容 → 执行导入流程
    │
    └── 文字输入
            │
            └── 识别资料内容 → 执行导入流程
```

### 2. 学科识别与分类

**学科识别方法**：
- **内容分析**：根据资料中的关键词、术语、公式等识别学科
- **用户指定**：用户明确指定资料所属学科
- **上下文推断**：根据对话上下文推断学科

**多学科资料处理**：
- 识别资料中涉及的所有学科
- 按学科分类处理和存储
- 生成跨学科索引

**支持学科**：语文、数学、英语、物理、化学、生物

**未支持学科处理**：遇到未支持学科的资料时，简单跳过即可，不进行处理；用户询问时，可告知不支持该学科。

### 3. 原始资料存储

按照全局约定，将原始资料存储到 `raw/YYYY/MM/` 目录：

- **存储时机**：在文件接收后**立即存储**，且**只存储一次**，后续处理步骤直接引用已存储的文件，不再重新拷贝
- **存储位置**：`raw/YYYY/MM/`
- **文件命名**：使用原始文件名存储，冲突时添加数字后缀
- **去重检查**：拷贝前必须检查 `raw/YYYY/MM/` 目录中是否已存在同名文件，若已存在则跳过拷贝，直接使用已存在的文件路径
- **路径引用**：在处理结果文件中引用正确的相对路径

### 4. 资料解析与完整内容保存

**解析内容**：
- **完整内容提取**：提取资料的全部内容，保留原始结构和格式
- **目录结构**：生成资料的完整目录结构（章节、小节等）
- **知识点提取**：提取资料中的核心知识点，建立知识点索引
- **关键信息**：提取重要概念、公式、定理、例题等

**结构化输出**：
- **扁平化存储**：所有资料在 `YYYY/MM/` 目录下扁平化存储，不建立 `<资料名称>` 子目录，目录结构清晰简洁
- **主文件**：创建主文件 `<YYYY-MM-DD>-<descriptive-slug>.md`，包含资料概述、目录结构和章节引用
- **子文件**：按章节拆分保存为独立子文件 `<YYYY-MM-DD>-<descriptive-slug>-<sub-slug>.md`，由主文件引用
- **包含 frontmatter 元数据**：主文件包含完整的 frontmatter 信息，记录章节列表和知识点索引
- **建立知识点索引**：在主文件中建立知识点与章节的映射关系

### 5. 按学科分类归档

**归档位置**：
- `subjects/<学科>/YYYY/MM/<YYYY-MM-DD>-<descriptive-slug>.md` — 资料主文件（包含目录结构和章节引用）
- `subjects/<学科>/YYYY/MM/<YYYY-MM-DD>-<descriptive-slug>-<sub-slug>.md` — 各章节内容子文件

**多学科资料处理**：
- 识别资料中涉及的所有学科
- 为每个学科创建独立的主文件和子文件
- 更新各学科的 `_index.md`

**Frontmatter 模板**（主文件）：
```yaml
---
date: YYYY-MM-DD
subject: [学科]
title: [资料名称]
ingest_type: textbook  # textbook/handout/notes/reference_material
source_path: [原始资料路径]
reference_type: [参考类型，如 standard_answer/model_essay]
total_pages: [总页数]
sections:
  - title: [章节标题1]
    file: [子文件名1]
    topics:
      - [知识点1]
      - [知识点2]
  - title: [章节标题2]
    file: [子文件名2]
    topics:
      - [知识点3]
related_topics:
  - [跨章节相关知识点1]
  - [跨章节相关知识点2]
---
```

**子文件命名规范**：
- 章节子文件按内容自然分隔，命名方式：`<YYYY-MM-DD>-<descriptive-slug>-<sub-slug>.md`
- 示例：`2026-07-14-algebra-functions-01-introduction.md`、`2026-07-14-algebra-functions-02-derivative-concept.md`

**主文件内容结构**：
```markdown
---
# [资料名称]

## 资料概述
[资料的简要介绍和背景信息]

## 目录
- [章节标题1](<YYYY-MM-DD>-<descriptive-slug>-<sub-slug>.md)
- [章节标题2](<YYYY-MM-DD>-<descriptive-slug>-<sub-slug>.md)

## 知识点索引
| 知识点 | 对应章节 |
|--------|----------|
| [知识点1] | [章节标题1] |
| [知识点2] | [章节标题1] |
```

**扁平化存储的好处**：
- **目录结构简洁**：所有资料在 `YYYY/MM/` 目录下扁平化存储，不需要建立嵌套子目录
- **方便检索**：通过文件名模式匹配快速定位资料，便于按日期、主题检索
- **避免重复处理**：一次性保存完整内容，后续任务可直接提取相关章节，无需重复处理原始资料
- **知识完整性**：保留资料的完整上下文，便于理解和学习

### 6. 索引更新

**更新内容**：
- 更新对应学科 `_index.md` 的资料索引
- 添加新导入的资料到「导入资料列表」，记录资料主文件路径
- 更新相关知识点的关联资料，建立知识点到资料章节的映射
- 记录资料包含的章节数量和知识点数量

**索引格式**：
- 在学科 `_index.md` 中添加「导入资料」章节
- 记录资料名称、日期、主文件路径、类型、章节数、知识点数等信息

**导入资料列表格式**（学科 `_index.md`）：
```markdown
## 导入资料
| 日期 | 资料名称 | 类型 | 路径 | 章节数 | 知识点 |
|------|----------|------|------|--------|--------|
| [YYYY-MM-DD] | [资料名称] | [textbook/handout/notes/reference] | [<YYYY-MM-DD>-<descriptive-slug>.md] | [章节数量] | [知识点列表] |
```

### 7. 操作概要记入 log 文件

每次执行资料导入流程后，必须将操作概要记入 `output/YYYY/MM/YYYY-MM-DD-log.md` 文件：

**记录内容**：
- **操作类型**：学习资料导入/多学科资料导入/参考材料导入
- **处理的文件**：上传的资料路径、识别的内容、涉及的学科
- **生成的文件**：创建的资料主文件和子文件路径（按学科列出）、章节文件数量
- **更新的索引**：更新的 `_index.md` 文件及其内容摘要
- **核心数据**：导入的资料数量、涉及的学科列表、总章节数、知识点数

**记录格式**：
```markdown
## 操作记录

### YYYY-MM-DD HH:MM:SS
- **操作类型**：学习资料导入
- **处理文件**：raw/2026/07/textbook_chapter1.jpg
- **生成文件**：subjects/数学/2026/07/2026-07-14-algebra-functions.md（主文件）、2026-07-14-algebra-functions-01-introduction.md 等（3个章节子文件）
- **更新索引**：subjects/数学/_index.md（添加导入资料：代数函数章节，3章，5个知识点）
- **核心数据**：导入1份资料，涉及1个学科，共3个章节，5个知识点
```

## 三、与其他 Workflow 的关系

### 调用关系

```
ingest_workflow (学习资料导入)
    │
    └── conversion_workflow (文件处理与内容转换，如适用)
            └── 文件存储到 raw/YYYY/MM/
```

### 数据传递

- **从 conversion_workflow 接收**：文件内容提取结果、文件存储路径、内容结构化数据
- **输出**：归档文件、索引更新、资料摘要

---

## 四、资料检索与复用

### 检索方式

- **按学科检索**：通过学科 `_index.md` 的资料索引检索，定位到资料主文件
- **按知识点检索**：通过知识点关联的资料章节检索，直接定位到相关章节子文件
- **按日期检索**：通过年月目录检索，通过文件名模式匹配定位资料
- **按章节检索**：通过资料主文件中的目录结构导航到具体章节子文件

### 与 learn 的集成

- learn 时可按学科检索导入的学习资料，定位到资料主文件
- 通过知识点索引直接提取相关章节内容（从章节子文件中），无需重复处理原始资料
- 将导入资料作为学习参考，支持知识点学习和复习
- 必要时通过引用链接寻找 raw 文件

### 完整资料文件结构示例

```
subjects/数学/2026/07/
├── 2026-07-14-algebra-functions.md                    # 资料主文件（目录结构和章节引用）
├── 2026-07-14-algebra-functions-01-introduction.md    # 第一章：引言
├── 2026-07-14-algebra-functions-02-derivative-concept.md  # 第二章：导数概念
├── 2026-07-14-algebra-functions-03-derivative-applications.md  # 第三章：导数应用
├── 2026-07-15-physics-formulas.md                     # 另一份资料主文件
└── 2026-07-15-physics-formulas-01-mechanics.md        # 第一章：力学公式
```

**优势**：
- **目录结构简洁**：所有资料在 `YYYY/MM/` 目录下扁平化存储，不需要建立嵌套子目录
- **结构化检索**：通过主文件快速了解资料结构和知识点覆盖
- **精准定位**：按章节子文件直接提取相关内容，无需处理整个文件
- **避免重复**：一次性保存完整内容，后续任务可直接引用

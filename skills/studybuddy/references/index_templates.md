# _index.md 模板定义

本文件定义 `~/.studybuddy/` 目录下各层级 `_index.md` 的模板格式。AI agent 在运行时根据模板创建和维护这些索引文件。

> **重要说明**：`_index.md` 仅追踪学生本人的学习数据（错题、正确率、薄弱点、考试成绩）。标准答案、作文范文、同学范文等参考材料存储在 `subjects/<学科>/YYYY/MM/` 目录下，文件名格式与普通错题一致，但在 frontmatter 中标记 `reference_type` 字段（`standard_answer`/`model_essay`/`classmate_essay`），此类文件不纳入 `_index.md` 的统计和索引。

## 3. _index.md 创建与更新规则

### 创建时机
1. **首次使用**：用户首次打招呼时，AI 创建 `~/.studybuddy/_index.md`（总览）和六科的 `subjects/<学科>/_index.md`（仅含静态模板内容：高频主题、高频考点、学习目标）
2. **新学科接触**：用户首次上传某学科作业时，如该科 `_index.md` 不存在，创建之

### 更新时机
1. **每次错题归档后**：
   - 更新对应学科的 `_index.md` 中的「薄弱点（盲区）」表：
     - 如知识点已存在：错误次数 +1，更新最近错误日期，状态设为 ⚠️ 待巩固
     - 如知识点不存在：新增一行，错误次数 = 1，状态设为 ⚠️ 待巩固
   - 更新「正确率趋势」
   - 更新「最近错题」列表（保留最近5条）
   - 更新文件顶部的「更新时间」
2. **每次复习完成后**：
   - 如连续 3 次答对同一知识点：将状态从 ⚠️ 待巩固 改为 ✅ 已掌握
3. **每日/每周自动汇总**：
   - 更新 root 层的 `_index.md`：汇总各学科正确率、薄弱点数量、最近错题日期、掌握度

### 薄弱点状态流转
```
新增（错误次数=1）→ ⚠️ 待巩固
连续3次答对 → ✅ 已掌握
已掌握后再次答错 → ⚠️ 待巩固（错误次数重置为1）
```

### 遗忘机制
- 薄弱点标记为 ✅ 已掌握后，再次出现错题时重新标记为 ⚠️ 待巩固
- 已掌握知识点超过 30 天未复习，自动降级为 ⚠️ 待巩固，提示用户复习

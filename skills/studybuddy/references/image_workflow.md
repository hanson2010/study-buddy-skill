# 图片处理与 OCR 识别 Workflow

本文件定义图片上传、处理、OCR 识别的通用流程，为作业批改、作文批改、古诗文学习等场景提供底层支持。

---

## 一、适用场景

| 场景 | 说明 | 覆盖题型 |
|------|------|----------|
| **学习资料导入** | PDF 学习资料的提取和归档 | 课本 PDF、讲义 PDF、笔记 PDF、参考材料 PDF |
| **古诗文学习** | 古诗文抄写、默写的识别和纠错 | 古诗词默写、文言文翻译、名句填空 |
| **试卷分析** | 整张试卷的识别和分析 | 所有题型 |
| **作业批改** | 作业题目识别和批改 | 选择题、填空题、解答题、计算题、证明题、判断题 |
| **作文批改** | 手写作文稿的识别和评价 | 议论文、记叙文、应用文、散文 |
| **论述题批改** | 文科论述题的识别和批改 | 政治论述题、历史材料题、地理分析题 |
| **实验题批改** | 理综实验题的识别和批改 | 物理实验、化学实验、生物实验 |
| **成绩分析** | 成绩分析报告和单科卷面分析报告的识别和分析 | 成绩分析报告、单科卷面分析报告 |

---

## 二、图片处理流程

### 0. 日期确定

**日期选择规则**：
- **优先从原始资料识别**：从图片内容中识别日期信息（如试卷日期、作业日期、考试日期、作文日期等）
- **次用当前时间**：当无法从原始资料中识别日期时，使用执行本工作的当前时间
- **文件夹创建**：根据确定的日期生成 `YYYY/MM` 路径；当文件夹不存在时自动创建，不得武断选择之前发现的文件夹

**日期识别优先级**：
1. 图片内容中的日期（如"2026年4月15日期中考试"）
2. 用户提供的日期信息
3. 当前系统时间

### 1. 文件接收
- 接收用户上传的图片文件和 PDF 文件
- 支持图片格式：jpg、jpeg、png、gif、bmp、webp
- 支持 PDF 格式：pdf
- 支持多张图片批量上传（如多页试卷、多页作文）
- 支持多页 PDF 文件（自动提取文本内容）

### 2. 图片预处理

#### 压缩规则
- 大于 2MB 的图片自动压缩至 2MB 以内
- 优先降低质量，必要时缩小尺寸
- 保持图片清晰度，确保 OCR 识别准确性

#### 存储规则
- **必须存入**：用户上传的任何资料（图片、文档等），无论使用哪个子命令（aim、learn、ingest、eval、feedback）或未明确指定子命令，都必须先存入 `raw/YYYY/MM/` 目录
- **文件命名**：使用原始文件名存储，不计算 MD5 哈希值
- **冲突处理**：当原文件名存在时，通过添加数字后缀避免原文件被覆盖（如 `photo.jpg` → `photo_1.jpg`）
- **路径引用**：处理结果文件中必须引用正确的相对路径（`raw/YYYY/MM/<文件名>`），如果文件名有修改，相应修改引用路径

### 3. PDF 处理（如适用）

#### PDF 文件识别
- 当用户上传 PDF 文件时，自动识别为 PDF 处理流程
- PDF 文件大小限制：不超过 50MB
- 支持多页 PDF，自动按页提取文本

#### PDF 文本提取
- 使用 PDF 解析库提取文本内容
- 保留文本结构（段落、换行、标题等）
- 记录页数和文件名信息

#### PDF 存储规则
- **必须存入**：PDF 文件存入 `raw/YYYY/MM/` 目录
- **文件命名**：使用原始文件名存储
- **冲突处理**：当原文件名存在时，通过添加数字后缀避免覆盖
- **文本导出**：提取的文本保存为 `<原始文件名>_text.md`，包含 frontmatter 元数据（`pdf_file`、`pages`）

### 4. 内容裁剪

#### 自动裁剪（首选）
- 使用大模型的视觉能力识别内容边界
- 根据场景自动分隔：
  - **作业/试卷**：按题目自然分隔（选择题、填空题、解答题等）
  - **作文**：按段落分隔，保持文章结构完整
  - **古诗文**：按诗句或段落分隔
- 每张图片裁剪出独立的内容单元

#### 手动裁剪（备选）
- 当自动裁剪不准确时，提示用户手动框选区域
- 支持用户标注题目序号、段落边界等

### 5. OCR 识别

#### 调用大模型 OCR
- 将裁剪后的图片交给大模型进行文字识别
- 根据不同场景和题型，识别不同内容：

**选择题**：
- 题目描述（题干、条件、要求）
- 选项（A/B/C/D/E 等）
- 用户手写答案
- 题号、分数、页码

**填空题**：
- 题目描述（完整的题干）
- 填空位置标记（如横线、括号）
- 用户手写答案（包括公式、符号）
- 题号、分数
- 多个填空时，按顺序识别

**解答题**：
- 题目描述（完整的题干和要求）
- 用户手写解答过程（包括公式、步骤、推导）
- 最终答案
- 题号、分数

**计算题**：
- 题目描述（完整的题干和已知条件）
- 用户手写计算过程（公式、代入数值、中间结果）
- 最终答案（含单位）
- 题号、分数

**证明题**：
- 题目描述（已知条件、求证结论）
- 用户手写证明过程（定理引用、逻辑推导）
- 题号、分数

**判断题**：
- 题目描述（判断命题）
- 用户手写判断结果（√/× 或 对/错）
- 题号、分数

**作文**：
- 作文标题
- 正文内容（按段落识别）
- 字数统计
- 错别字标记
- 涂改痕迹

**古诗文**：
- 诗词题目、作者、朝代
- 正文内容（按句识别）
- 用户默写内容
- 错别字标记
- 标点符号

**论述题**：
- 题目描述（问题、材料、要求）
- 用户手写论述内容（论点、论据、论证）
- 题号、分数

**实验题**：
- 题目描述（实验目的、器材、步骤）
- 用户手写答案（实验现象、数据记录、结论）
- 图表、数据表格
- 题号、分数

**成绩分析报告**：
- 考试基本信息（考试名称、日期、年级、班级）
- 各科成绩（分数、满分、排名、班级平均分、年级平均分）
- 总分、总分排名、进步/退步情况
- 老师评语
- 各题型得分情况（单科卷面分析报告）
- 各知识点得分情况（单科卷面分析报告）
- 错误类型分布（单科卷面分析报告）

### 6. 内容理解
- 将 OCR 识别结果交给大模型进行语义理解
- 根据场景提取关键信息：
  - **作业批改**：题目类型、知识点、难度等级
  - **作文批改**：立意、结构、语言、素材
  - **古诗文学习**：体裁、主题、意象、情感
- 判断内容是否完整，是否需要用户补充信息

## 五、与各业务流程的衔接

### 业务流程总览

```
文件上传 → [image_workflow.md] 文件处理（图片/OCR 或 PDF/文本提取） → 业务流程分发
                                                                 │
                            ┌──────────────┬───────────┬─────────┼───────────────┬──────────────────────┐
                            ▼              ▼           ▼         ▼               ▼                      ▼
                   [作业批改模块] [feedback_] [essay_] [classical_] [ingest_]
                   (各题型)      workflow    workflow   workflow    workflow
                                  (成绩反馈) (作文批改) (古诗文学习) (学习资料导入)
                            │              │           │           │               │
                            ▼              ▼           ▼           ▼               ▼
                   错题归档、薄弱点更新  成绩记录更新 作文薄弱点更新 默写薄弱点更新 资料归档、索引更新
```

### 作业批改流程
```
图片上传 → 图片处理与 OCR 识别 → 作业批改模块 → 错题归档、薄弱点更新
```

**题型支持**：选择题、填空题、解答题、计算题、证明题、判断题、论述题、实验题

**数据传递**：OCR 识别结果包含完整的题目描述、用户答案、结构化信息（`question_type`、`options`、`solution_steps` 等）

### 作文批改流程

**核心原则**：作文批改必须先经过 OCR 识别，将图片内容转换为文字后，再进入作文批改流程。

```
图片上传 → [image_workflow.md] 图片处理与 OCR 识别 → [essay_workflow.md] 作文批改模块 → 作文薄弱点更新
```

**OCR 输出要求**：
- 作文格式 JSON（见前文「作文格式」）
- 必须包含 `title`（标题）、`paragraphs`（段落内容）、`word_count`（字数）、`essay_type`（作文类型）
- 可选包含 `errors`（错别字标记）

**与 essay_workflow.md 的衔接规则**：
1. **前置条件**：`image_workflow.md` 必须完成 OCR 识别，生成作文格式的结构化数据
2. **数据传递**：OCR 结果作为 `essay_workflow.md` 的输入，包含完整的作文文本和元数据
3. **引用记录**：作文文件中记录图片引用路径，方便回溯查看手写原稿
4. **错误处理**：OCR 识别失败或不完整时，提示用户补充缺失内容，不进入作文批改流程

### 古诗文学习流程
```
图片上传 → 图片处理与 OCR 识别 → 古诗文学习模块 → 默写薄弱点更新
```

**数据传递**：OCR 识别结果包含诗词题目、作者、朝代、正文内容、用户默写内容、错别字标记

### 成绩反馈流程

**核心原则**：成绩反馈必须先经过 OCR 识别，将图片内容转换为结构化数据后，再进入成绩反馈流程。

```
图片上传 → [image_workflow.md] 图片处理与 OCR 识别 → [feedback_workflow.md] 成绩反馈模块 → 成绩记录更新
```

**OCR 输出要求**：
- 成绩分析报告格式 JSON 或单科卷面分析报告格式 JSON（见前文对应格式）
- 必须包含 `report_type`（报告类型）、`exam_name`（考试名称）、`exam_date`（考试日期）
- 成绩分析报告必须包含 `scores`（各科成绩）、`total_score`（总分）、`total_ranking`（总分排名）
- 单科卷面分析报告必须包含 `subject`（学科）、`score`（分数）、`knowledge_points`（知识点得分）

**与 feedback_workflow.md 的衔接规则**：
1. **前置条件**：`image_workflow.md` 必须完成 OCR 识别，生成成绩分析报告格式的结构化数据
2. **数据传递**：OCR 结果作为 `feedback_workflow.md` 的输入，包含完整的成绩数据和元数据
3. **引用记录**：报告文件中记录图片引用路径，方便回溯查看原始报告
4. **错误处理**：OCR 识别失败或数据不完整时，提示用户补充缺失内容，不进入成绩分析流程

### 学习资料导入流程

**核心原则**：学习资料导入支持图片和 PDF 两种格式，图片经过 OCR 识别，PDF 直接提取文本内容。

```
文件上传 → [image_workflow.md] 文件处理（图片/OCR 或 PDF/文本提取） → [ingest_workflow.md] 学习资料导入模块 → 资料归档、索引更新
```

**文件处理要求**：
- **图片格式**：经过 OCR 识别，提取文本内容
- **PDF 格式**：直接提取文本内容，生成 `<文件名>_text.md` 文件
- **存储规则**：原始文件存入 `raw/YYYY/MM/` 目录

**与 ingest_workflow.md 的衔接规则**：
1. **前置条件**：`image_workflow.md` 必须完成文件处理，生成文本内容和存储路径
2. **数据传递**：处理结果作为 `ingest_workflow.md` 的输入，包含文件路径、文本内容、页数（PDF）等信息
3. **引用记录**：归档文件中记录原始文件引用路径，方便回溯查看原始资料
4. **错误处理**：文件处理失败时，提示用户重新上传或手动输入内容

### 数据传递规范

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| `content_id` | 内容唯一标识 | 是 |

## 六、错误处理

### OCR 识别失败
- 提示用户图片模糊，请求重新上传
- 提供手动输入内容的选项
- 识别不完整时，提示用户补充缺失内容

### 内容裁剪失败
- 提示用户手动框选内容区域
- 支持用户标注题目序号、段落边界等

### 图片格式不支持
- 提示用户转换图片格式（推荐 jpg、png）

### PDF 处理失败
- 提示用户 PDF 文件损坏或无法读取
- 提供手动输入内容的选项
- 文件过大时提示用户压缩文件

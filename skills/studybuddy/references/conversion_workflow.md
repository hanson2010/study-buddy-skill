# 文件接收与内容转换 Workflow（conversion_workflow）

本文件定义各类文件（图片、PDF、Office）的上传、存储、内容提取与结构化的通用流程，为作业批改、作文批改、古诗文学习、学习资料导入、成绩分析等场景提供底层支持。

---

## 一、适用场景

| 场景 | 说明 |
|------|------|
| **学习资料导入** | PDF/Office 学习资料的提取和归档 |
| **古诗文学习** | 古诗文抄写、默写的识别和纠错 |
| **试卷分析** | 整张试卷的识别和分析 |
| **作业批改** | 作业题目识别和批改 |
| **作文批改** | 手写作文稿的识别和评价 |
| **论述题批改** | 文科论述题的识别和批改 |
| **实验题批改** | 理综实验题的识别和批改 |
| **成绩分析** | 成绩分析报告和单科卷面分析报告的识别和分析 |

## 三、阶段1：文件接收与类型分发

### 3.1 文件接收
- 接收用户上传的图片文件、PDF 文件和 Office 文件
- 支持图片格式：jpg、jpeg、png、gif、bmp、webp
- 支持 PDF 格式：pdf
- 支持 Office 格式：docx、xlsx、xls、pptx（.doc、.ppt 格式不支持）
- 支持多张图片批量上传（如多页试卷、多页作文）
- 支持多页 PDF 文件和 Office 文件

### 3.2 日期确定

**日期选择规则**：
- **优先从原始资料识别**：从文件内容中识别日期信息（如试卷日期、作业日期、考试日期等）
- **次用当前时间**：当无法从原始资料中识别日期时，使用执行本工作的当前时间
- **文件夹创建**：根据确定的日期生成 `YYYY/MM` 路径；当文件夹不存在时自动创建

**日期识别优先级**：
1. 文件内容中的日期（如"2026年4月15日期中考试"）
2. 用户提供的日期信息
3. 当前系统时间

### 3.3 类型分发逻辑

| 文件类型 | 扩展名 | 分发目标 |
|----------|--------|----------|
| 图片 | .jpg, .jpeg, .png, .gif, .bmp, .webp | 调用 image_processor.py |
| PDF | .pdf | 调用 pdf_processor.py |
| Office Word | .docx | 调用 office_processor.py |
| Office Excel | .xlsx, .xls | 调用 office_processor.py |
| Office PowerPoint | .pptx | 调用 office_processor.py |
| 不支持 | .doc, .ppt | 提示用户转换格式 |

## 四、阶段2：脚本执行

### 4.1 通用存储规则

所有文件在脚本执行时遵循以下存储规则：
- **存储时机**：文件接收后立即存储，且只存储一次，后续处理步骤直接引用已存储的文件
- **存储目录**：存入 `raw/YYYY/MM/` 目录
- **文件命名**：使用原始文件名存储，不计算 MD5 哈希值
- **冲突处理**：当原文件名存在时，通过添加数字后缀避免覆盖（如 `photo.jpg` → `photo_1.jpg`）
- **去重检查**：拷贝前检查 `raw/YYYY/MM/` 目录中是否已存在同名文件，若已存在则跳过拷贝

### 4.2 image_processor.py（图片处理器）

**调用时机**：当用户上传图片文件时

**功能**：
- **图片压缩**：大于 2MB 的图片自动压缩至 2MB 以内，优先降低质量，必要时缩小尺寸，保持 OCR 识别准确性
- **文件存储**：将图片存储到 `raw/YYYY/MM/` 目录
- **路径返回**：返回相对路径 `raw/YYYY/MM/<文件名>`

**注意**：`image_processor.py` **不执行 OCR**，仅负责图片压缩和存储。OCR 由阶段3 的 LLM 视觉能力完成。

**调用方式**：
```powershell
& "$env:STUDYBUDDY_DATA_DIR\.venv\Scripts\python.exe" skills/studybuddy/scripts/image_processor.py <图片文件>
```

### 4.3 pdf_processor.py（PDF 处理器）

**调用时机**：当用户上传 PDF 文件时

**功能**：
- **文本提取**：使用 pymupdf4llm 提取 PDF 文本内容，生成 Markdown 格式
- **文件存储**：将 PDF 存储到 `raw/YYYY/MM/` 目录
- **文本导出**：提取的文本保存为 `<原始文件名>_text.md`，包含 frontmatter 元数据（`pdf_file`、`pages`）
- **路径返回**：返回相对路径 `raw/YYYY/MM/<文件名>`

**调用方式**：
```powershell
& "$env:STUDYBUDDY_DATA_DIR\.venv\Scripts\python.exe" skills/studybuddy/scripts/pdf_processor.py <PDF文件>
```

### 4.4 office_processor.py（Office 处理器）

**调用时机**：当用户上传 Office 文件（docx、xlsx、xls、pptx）时

**功能**：
- **DOCX**：提取段落文本和表格内容，保留标题层级和列表结构，转换为 Markdown
- **XLSX/XLS**：按工作表提取数据，转换为 Markdown 表格格式
- **PPTX**：按幻灯片提取文本内容，保留幻灯片边界
- **文件存储**：将 Office 文件存储到 `raw/YYYY/MM/` 目录
- **文本导出**：提取的文本保存为 `<原始文件名>_text.md`，包含 frontmatter 元数据（`office_file`、`file_type`、`paragraphs`/`sheets`/`slides`）
- **路径返回**：返回相对路径 `raw/YYYY/MM/<文件名>`

**调用方式**：
```powershell
& "$env:STUDYBUDDY_DATA_DIR\.venv\Scripts\python.exe" skills/studybuddy/scripts/office_processor.py <Office文件>
```

## 五、阶段3：文本提取评估

### 5.1 评估目的

脚本提取文本后，需要判断文本质量，决定是否需要额外的 OCR 处理。

### 5.2 评估规则

| 文件类型 | 评估条件 | 是否需要 OCR |
|----------|----------|--------------|
| **图片文件** | - | ✅ 总是需要 OCR |
| **PDF 文件** | 提取的文本为空或过少（< 100 字符） | ✅ 需要 OCR（扫描版 PDF） |
| **PDF 文件** | 提取的文本充足（≥ 100 字符） | ❌ 跳过 OCR |
| **PDF 文件** | 文本充足但包含图片区域 | ⚠️ 仅对图片区域 OCR |
| **DOCX 文件** | 提取的文本为空或过少 | ✅ 需要 OCR（图片为主的文档） |
| **DOCX 文件** | 文本充足但包含嵌入图片 | ⚠️ 仅对嵌入图片 OCR |
| **XLSX/XLS** | - | ❌ 通常不需要 OCR |
| **PPTX 文件** | 提取的文本为空或过少 | ✅ 需要 OCR（图片为主的幻灯片） |
| **PPTX 文件** | 文本充足但包含图片 | ⚠️ 仅对图片区域 OCR |

### 5.3 扫描版 PDF 处理

当判断为扫描版 PDF 时：
1. 使用 PDF 工具将每一页转换为图片
2. 对每张图片执行 OCR
3. 将 OCR 结果与脚本提取的文本合并

### 5.4 Office 嵌入图片处理

当 Office 文档包含嵌入图片时：
1. 提取文档中的嵌入图片
2. 对每张图片执行 OCR
3. 将 OCR 结果插入到对应位置的文本中

## 六、阶段4：OCR 识别

### 6.1 OCR 方式

所有 OCR 通过 **LLM 视觉能力**完成，不使用独立的 OCR 脚本或库。

### 6.2 内容裁剪

#### 自动裁剪（首选）
- 使用大模型的视觉能力识别内容边界
- 根据场景自动分隔：
  - **作业/试卷**：按题目自然分隔（选择题、填空题、解答题等）
  - **作文**：按段落分隔，保持文章结构完整
  - **古诗文**：按诗句或段落分隔
- 每张图片裁剪出独立的内容单元

#### 手动裁剪（备选）
- 当自动裁剪不准确时，提示用户手动框选区域
- 支持用户标注题目序号、段落边界等

### 6.3 OCR 场景分类与识别字段

**选择题**：题目描述（题干、条件、要求）、选项（A/B/C/D/E 等）、用户手写答案、题号、分数、页码

**填空题**：题目描述（完整的题干）、填空位置标记（如横线、括号）、用户手写答案（包括公式、符号）、题号、分数、多个填空时按顺序识别

**解答题**：题目描述（完整的题干和要求）、用户手写解答过程（包括公式、步骤、推导）、最终答案、题号、分数

**计算题**：题目描述（完整的题干和已知条件）、用户手写计算过程（公式、代入数值、中间结果）、最终答案（含单位）、题号、分数

**证明题**：题目描述（已知条件、求证结论）、用户手写证明过程（定理引用、逻辑推导）、题号、分数

**判断题**：题目描述（判断命题）、用户手写判断结果（√/× 或 对/错）、题号、分数

**作文**：作文标题、正文内容（按段落识别）、字数统计、错别字标记、涂改痕迹

**古诗文**：诗词题目、作者、朝代、正文内容（按句识别）、用户默写内容、错别字标记、标点符号

**论述题**：题目描述（问题、材料、要求）、用户手写论述内容（论点、论据、论证）、题号、分数

**实验题**：题目描述（实验目的、器材、步骤）、用户手写答案（实验现象、数据记录、结论）、图表、数据表格、题号、分数

**成绩分析报告**：考试基本信息（考试名称、日期、年级、班级）、各科成绩（分数、满分、排名、班级平均分、年级平均分）、总分、总分排名、进步/退步情况、老师评语、各题型得分情况、各知识点得分情况、错误类型分布

## 十、返回数据结构

`conversion_workflow` 完成文件处理后，向调用方返回统一的数据结构。调用方根据 `scene_type` 字段自行解析场景特定内容。

### 通用返回字段

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| `content_id` | 内容唯一标识 | 是 |
| `file_path` | 文件存储路径（`raw/YYYY/MM/` 下的相对路径） | 是 |
| `file_type` | 文件类型（`image`/`pdf`/`office`） | 是 |
| `extracted_text` | 完整提取文本（OCR 识别结果或脚本提取的文本） | 是 |
| `scene_type` | 场景类型，用于业务分发 | 是 |
| `metadata` | 场景特定的结构化数据，具体字段由各业务工作流定义 | 是 |

**返回方式**：以 JSON 格式返回，调用方根据 `scene_type` 自行解析 `metadata` 中的场景特定字段。

## 十一、错误处理

### OCR 识别失败
- 提示用户图片模糊，请求重新上传
- 提供手动输入内容的选项
- 识别不完整时，提示用户补充缺失内容

### 内容裁剪失败
- 提示用户手动框选内容区域
- 支持用户标注题目序号、段落边界等

### 文件格式不支持
- 提示用户转换图片格式（推荐 jpg、png）
- 提示用户将 .doc/.ppt 转换为 .docx/.pptx

### PDF 处理失败
- 提示用户 PDF 文件损坏或无法读取
- 提供手动输入内容的选项
- 文件过大时提示用户压缩文件

### 脚本执行失败
- 检查虚拟环境和依赖是否安装
- 提示用户运行安装命令

# 知识点学习工作流（learn_workflow）

本文件定义通用知识点学习与巩固的处理流程，支持多种学习场景，在必要时调用 classical_workflow。

---

## 一、适用场景

- **知识点学习**：用户主动学习某个知识点
- **薄弱点复习**：针对历史薄弱点进行专项复习
- **古诗文学习**：学习古诗词、文言文（调用 classical_workflow）
- **概念理解**：深入理解某个学科概念
- **公式推导**：学习数学/物理公式的推导过程
- **实验原理**：学习化学/生物实验的原理和操作

---

## 二、处理流程

### 1. 学习需求识别

**输入类型识别**：
- **明确知识点**：用户指定要学习的知识点（如"学习导数的应用"）
- **薄弱点复习**：用户请求复习薄弱点（如"复习我的薄弱点"）
- **古诗文学习**：用户请求学习古诗文（如"学习《登高》"）
- **概念理解**：用户请求理解某个概念（如"什么是牛顿第二定律"）

**内容类型识别**：
- **普通知识点**：数学公式、物理定律、化学方程式、生物概念等
- **古诗文**：古诗词、文言文（调用 classical_workflow）
- **实验内容**：实验原理、操作步骤、数据处理等

**路由逻辑**：
```
用户输入
    │
    ├── 明确知识点/概念理解 → 执行通用学习流程
    │
    ├── 薄弱点复习 → 读取 _index.md 薄弱点 → 执行通用学习流程
    │
    └── 古诗文学习 → 调用 classical_workflow
```

**未支持学科处理**：本技能仅支持六科（语文、数学、英语、物理、化学、生物）。识别学习需求所属学科后，若为未支持学科，简单跳过即可，不进行学习处理；用户询问时，可告知不支持该学科。

### 2. 学情数据读取

**读取内容**：
- **profile.md**：学生基础信息、教材版本、学习偏好
- **_index.md**：薄弱点表、正确率趋势、最近错题、导入资料索引
- **subjects/<学科>/YYYY/MM/**：相关错题记录和导入资料
- **导入资料检索**：按学科检索 `subjects/<学科>/YYYY/MM/` 下已导入的学习资料，通过主文件了解资料结构和知识点覆盖，按章节子文件直接提取相关内容，必要时通过引用链接寻找 raw 文件

**个性化调整**：
- 根据学生年级和教材版本调整讲解深度
- 根据薄弱点状态调整复习重点
- 根据学习偏好调整讲解方式

### 3. 视频推荐

在进行详细知识点讲解前，先搜寻并推荐 1-2 个国内平台的优质概念/方法类微课视频，或提供高精确度搜索关键词，帮助学生快速建立背景理论框架。

**推荐流程**：
- 搜索相关知识点讲解视频（优先使用 `web_search` 工具实时搜索）
- 推荐 1-2 个优质视频，提供视频标题、链接、推荐理由

**降级策略**：
- 无 `web_search` 工具时提供搜索关键词
- 引导用户在 B站/国家中小学智慧教育平台搜索

### 4. 知识点讲解与交互确认

**讲解流程**：
1. **概念引入**：用生活实例或类比引入知识点
2. **核心定义**：给出准确的定义和公式
3. **原理推导**：讲解公式/定理的推导过程
4. **例题演示**：通过例题演示应用方法
5. **易错点提醒**：指出常见错误和注意事项

**可视化讲解**：
- 使用图形、图表辅助理解
- 使用 ASCII 图表或 Markdown 表格
- 几何题用示意图、物理题用受力分析图、化学题用反应机理图

> [!IMPORTANT]
> **交互式断点**：知识点讲解完成后，**必须暂停并询问学生是否完全搞懂该知识点，或是否需要调整讲解角度/难度**。在学生未确认理解或主动要求前，禁止在对话中显式输出巩固练习（第 6 部分）。

### 5. 古诗文学习（如适用）

当识别为古诗文学习时，调用 [classical_workflow.md](classical_workflow.md)：

- **初读感知**：整体感知诗文内容和情感
- **字词解析**：逐字逐词解释重点字词
- **句意翻译**：翻译诗文大意
- **内容赏析**：分析诗文的艺术特色和思想内涵
- **情境联想**：通过想象加深理解
- **记忆巩固**：使用多种记忆技巧帮助背诵

### 6. 练习巩固

**练习生成**：
- 根据知识点生成梯度练习题
- 基础巩固 → 提高强化 → 挑战拔高
- 单次不超过 8 题

**练习形式**：
- **选择题**：检验概念理解
- **填空题**：检验公式记忆
- **解答题**：检验应用能力
- **默写题**：检验古诗文背诵

### 7. 学习效果评估

**评估方式**：
- **即时反馈**：练习完成后立即批改并给出解析
- **正确率统计**：统计本次练习的正确率
- **薄弱点更新**：根据练习结果更新薄弱点状态

**状态更新**：
- 连续 3 次答对 → 标记为 ✅ 已掌握
- 错误率较高 → 标记为 ⚠️ 待巩固
- 超过 30 天未复习 → 标记为 ⚠️ 待巩固

### 8. 学习记录归档

**归档内容**：
- **学习内容**：知识点名称、讲解摘要
- **练习记录**：题目内容、答案、正确率
- **视频推荐**：推荐的视频链接

**归档位置**：
- `subjects/<学科>/YYYY/MM/<YYYY-MM-DD>-<topic-slug>-learning.md`

**Frontmatter 模板**：
```yaml
---
date: YYYY-MM-DD
subject: [学科]
topic: [知识点]
learning_type: [学习类型]
correct_rate: [正确率]
status: learned  # learned/reviewing/weak
videos:
  - [视频标题]: [视频链接]
---
```

## 三、与其他 Workflow 的关系

### 调用关系

```
learn_workflow (通用知识点学习)
    │
    └── classical_workflow (古诗文学习)
            └── 古诗文特定的学习流程
```

### 数据传递

- **传递给 classical_workflow**：古诗文名称、学习目标
- **输出**：学习记录、练习结果、薄弱点更新

## 四、学习策略

### 1. 循序渐进策略

**学习顺序**：
1. **基础概念**：理解基本定义和公式
2. **简单应用**：通过简单例题掌握基本方法
3. **综合应用**：解决综合性问题
4. **拓展延伸**：探索知识点的深层含义和应用

### 2. 间隔重复策略

**复习周期**：
- 第 1 次学习后：1 天复习
- 第 2 次复习后：3 天复习
- 第 3 次复习后：7 天复习
- 第 4 次复习后：14 天复习

**复习方式**：
- 快速回顾核心概念
- 做 2-3 道练习题检验记忆

### 3. 错题驱动策略

**学习路径**：
1. **识别薄弱点**：从 `_index.md` 获取薄弱点列表
2. **针对性学习**：针对薄弱点进行专项学习
3. **练习巩固**：生成针对性练习题
4. **评估效果**：检查正确率是否提升

### 4. 操作概要记入 log 文件

每次执行知识点学习流程后，必须将操作概要记入 `output/YYYY/MM/YYYY-MM-DD-log.md` 文件：

**记录内容**：
- **操作类型**：知识点学习/薄弱点复习/古诗文学习/概念理解
- **学习内容**：学习的知识点名称、讲解摘要
- **练习记录**：生成的练习题数量、正确率
- **视频推荐**：推荐的视频链接
- **更新的索引**：更新的 `_index.md` 文件及其内容摘要

**记录格式**：
```markdown
## 操作记录

### YYYY-MM-DD HH:MM:SS
- **操作类型**：知识点学习
- **学习内容**：导数的应用
- **练习记录**：生成5题，正确率80%
- **视频推荐**：[导数应用专题讲解](链接)
- **更新索引**：subjects/数学/_index.md（更新薄弱点状态）
```

## 五、使用示例

### 输入示例1：学习知识点
```text
我想学习导数的应用。
```

### 输入示例2：复习薄弱点
```text
帮我复习数学的薄弱点。
```

### 输入示例3：学习古诗文
```text
学习《登高》这首诗。
```

### 输入示例4：理解概念
```text
什么是光合作用？请详细解释一下。
```

### 输入示例5：学习公式推导
```text
帮我推导一下勾股定理。
```

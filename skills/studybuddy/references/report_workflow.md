# 学情追踪工作流（report_workflow）

本文件定义学情追踪的处理流程，自动统计并可视化学习数据，生成学情报告保存到 `output/` 目录，为学生和家长提供全面的学习分析。

---

## 一、学情追踪目标

- **数据可视化**：将分散的学习数据汇总成直观的报告
- **趋势分析**：识别学习趋势，发现进步或退步的学科和知识点
- **薄弱点预警**：预测可能的薄弱点，提前干预
- **个性化建议**：基于数据给出针对性的复习建议
- **家长沟通**：生成适合家长阅读的学习报告

---

## 二、数据来源

### 1. _index.md 文件
- **Root _index.md**：全局薄弱点、历次考试成绩、各科状态汇总
- **学科 _index.md**：各学科薄弱点表、正确率趋势、最近错题

### 2. subjects/ 目录
- 各学科的错题记录（按科目+年月归档）
- 各学科的练习记录

### 3. raw/ 目录
- 日志文件（`YYYY-MM-DD-log.md`）
- 原始图片资源

### 4. profile.md
- 学生基础信息（姓名、年级、文理科、教材版本）

---

## 三、报告类型

### 1. 周报（weekly）

**适用场景**：每周自动生成，总结一周的学习情况

**包含内容**：
- 本周学习天数和总时长
- 本周批改题目数量和正确率趋势
- 本周新增/改进薄弱点
- 本周错题类型分布
- 各科表现对比
- 下周复习建议

### 2. 月报（monthly）

**适用场景**：每月自动生成，总结一个月的学习情况

**包含内容**：
- 本月学习天数和总时长
- 本月批改题目数量和正确率趋势
- 本月薄弱点变化（新增、改进、已掌握）
- 各科表现对比和排名变化
- 知识点掌握度分析
- 下月复习计划

### 3. 专项报告（special）

**适用场景**：针对特定需求生成的专项分析报告

**包含类型**：
- **薄弱点专项报告**：分析所有薄弱点的详细情况
- **考试对比报告**：对比多次考试的成绩变化
- **学科分析报告**：深入分析某一学科的学习情况

---

## 四、学情追踪流程

### 0. 数据充足性检查（仅自动触发时）

**适用场景**：周报、月报的自动触发

**检查规则**：
- **周报数据不足条件**：统计周期内（7天）批改题目数 < 5 道 或 学习天数 < 2 天
- **月报数据不足条件**：统计周期内（30天）批改题目数 < 10 道 或 学习天数 < 5 天

**处理方式**：
- 当数据不足时，**跳过本次报告生成**，不输出报告文件
- 记录跳过原因到日志文件中
- 下次触发时重新检查

**手动触发**：用户手动请求生成报告时，不受数据充足性限制，正常生成报告

### 1. 数据收集与整理

**数据收集**：
- 读取 root `_index.md` 获取全局数据
- 读取各学科 `_index.md` 获取学科数据
- 读取 `subjects/<学科>/YYYY/MM/` 获取详细错题记录
- 读取 `raw/YYYY/MM/YYYY-MM-DD-log.md` 获取日志记录

**数据整理**：
- 按时间维度汇总（日、周、月）
- 按学科维度汇总
- 按知识点维度汇总
- 按错误类型维度汇总

### 2. 数据统计与分析

**正确率统计**：
- 各学科正确率计算
- 各知识点正确率计算
- 正确率趋势分析（环比、同比）

**错误类型统计**：
- 概念不清、计算失误、审题错误、方法不当等类型的分布
- 各错误类型的变化趋势

**薄弱点分析**：
- 薄弱点数量统计
- 薄弱点变化分析（新增、改进、已掌握）
- 薄弱点优先级排序

**学习时长统计**：
- 每日学习时长
- 每周学习天数和总时长
- 各学科学习时长分配

### 3. 趋势分析

**正确率趋势**：
- 近7天/30天正确率变化
- 各科正确率对比趋势

**薄弱点趋势**：
- 薄弱点数量变化趋势
- 各知识点薄弱点状态变化

**排名趋势**：
- 总分排名变化趋势
- 各科排名变化趋势

### 4. 个性化建议生成

**基于薄弱点**：
- 针对薄弱知识点推荐复习内容
- 推荐相关视频资源
- 生成针对性练习题

**基于趋势**：
- 针对下降趋势的学科建议加强练习
- 针对上升趋势的学科建议保持并拓展

**基于学习时长**：
- 建议合理分配各学科学习时间
- 建议增加薄弱学科的学习时间

### 5. 报告生成与输出

**报告格式**：Markdown 文件

**存储位置**：`output/YYYY/MM/`

**文件名格式**：`YYYY-MM-DD-report-<report_type>.md`（如 `2026-07-12-report-weekly.md`）

**输出内容**：
- 报告标题和生成时间
- 核心数据概览（表格形式）
- 详细分析（文字描述）
- 可视化图表（ASCII 图表或 Markdown 表格）
- 个性化建议
- 下一步学习计划

### 操作概要记入 log 文件

每次执行学情追踪流程后，必须将操作概要记入 `raw/YYYY/MM/YYYY-MM-DD-log.md` 文件：

**记录内容**：
- **操作类型**：学情周报/学情月报/专项报告
- **报告类型**：weekly/monthly/special
- **生成的文件**：创建的报告文件路径
- **统计周期**：分析的起止日期
- **核心数据**：学习天数、学习时长、批改题目数量、正确率、薄弱点数量

**记录格式**：
```markdown
## 操作记录

### YYYY-MM-DD HH:MM:SS
- **操作类型**：学情周报
- **报告类型**：weekly
- **生成文件**：output/2026/07/2026-07-12-report-weekly.md
- **统计周期**：2026-07-06 至 2026-07-12
- **核心数据**：学习5天，时长12小时，批改45题，正确率78%，薄弱点8个
```

## 七、与 _index.md 的集成规则

### 数据同步
- **报告生成后**：不直接修改 `_index.md`，报告作为独立输出文件保存
- **薄弱点更新**：薄弱点的状态变化仍通过 `_index.md` 记录，报告仅读取数据

### 报告引用
- 在 root `_index.md` 的「最近报告」部分记录生成的报告路径
- 方便快速查阅历史报告

## 八、报告生成时机

| 报告类型 | 生成时机 | 触发方式 | 数据充足性检查 |
|----------|----------|----------|----------------|
| 周报 | 每周日 22:00 | 自动触发 | ✅ 数据不足时跳过 |
| 月报 | 每月最后一天 22:00 | 自动触发 | ✅ 数据不足时跳过 |
| 专项报告 | 用户请求时 | 手动触发 | - |
